## Bugs Found and Fixed

### Critical

| File | Issue | Fix |
|------|-------|-----|
| `animation/animation_sampler.cpp` | `readVec3Cubic`/`readQuatCubic` missing bounds checks — OOB read on malformed CUBICSPLINE data | Added bounds validation matching `readVec3`/`readQuat` |

### Medium

| File | Issue | Fix |
|------|-------|-----|
| `core/logger.cpp` | `openLogFile()` doesn't close existing file; `create_directories` can throw unhandled; `localtime_r` is POSIX-only | Close before reopen; wrap in try/catch; add `#ifdef _WIN32` for `localtime_s` |

| `shaders/bloom_bright.frag.glsl` | No sanitization — Inf from scene shader propagates through bloom chain | Added `clamp(color, vec3(0.0), vec3(65504.0))` before luminance calculation |

## Security Hardening

### CMake (engine/CMakeLists.txt)

Added OpenSSF-recommended flags:
- `-Wsign-conversion` — catch implicit signed/unsigned conversion
- `-Wimplicit-fallthrough` — catch missing `break` in switch
- `-Werror=format-security` — error on insecure format strings
- `-ftrivial-auto-var-init=zero` — zero-initialize stack variables
- `-fno-delete-null-pointer-checks` — preserve null checks in optimized code
- `-fno-strict-overflow` — treat signed overflow as defined
- `-Wl,--as-needed` — eliminate unnecessary library dependencies
- `_FORTIFY_SOURCE=3` (upgraded from 2) — dynamic buffer size checks (GCC 12+)
- `_GLIBCXX_ASSERTIONS` — runtime precondition checks on libstdc++ containers

**Source:** [OpenSSF Compiler Options Hardening Guide](https://best.openssf.org/Compiler-Hardening-Guides/Compiler-Options-Hardening-Guide-for-C-and-C++.html)

### Audio Input Validation (audio/audio_clip.cpp)

Added metadata validation for all four audio decoders (WAV, MP3, FLAC, OGG):
- Maximum frame count: 86.4M (~30 min at 48kHz)
- Channel count: 1-8
- Sample rate: 8,000-192,000 Hz
- `size_t` arithmetic for buffer allocation to prevent integer overflow

## Experimental Features Research

### Recommended for Investigation

1. **Bindless Textures** (`GL_ARB_bindless_texture`)
   - Eliminates texture binding overhead for scenes with many materials
   - Supported on AMD RDNA2 and NVIDIA
   - Significant draw call reduction for architectural walkthroughs
   - **Sources:** [Bindless Tutorial](https://ktstephano.github.io/rendering/opengl/bindless), [Safety Guide](https://gist.github.com/JuanDiegoMontoya/55482fc04d70e83729bb9528ecdc1c61)

2. **XeGTAO** (Intel, MIT license)
   - State-of-the-art SSAO: 3-pass compute pipeline (depth prefilter → GTAO → bilateral denoise)
   - ~0.56ms at 1080p on RTX 2060
   - **Source:** https://github.com/GameTechDev/XeGTAO

3. **Kulla-Conty Energy Compensation** for IBL
   - Fixes dark appearance at high roughness (multi-scattering energy loss)
   - Small 2D LUT addition to existing IBL pipeline
   - **Sources:** [Bruno Opsenica](https://bruop.github.io/ibl/), [Godot PR #103934](https://github.com/godotengine/godot/pull/103934)

4. **Persistent Mapped Buffers** (core in OpenGL 4.4)
   - Triple-buffered streaming with `GL_MAP_PERSISTENT_BIT | GL_MAP_COHERENT_BIT`
   - Eliminates map/unmap overhead for particles, per-frame uniforms, dynamic vertices
   - **Source:** [C++ Stories](https://www.cppstories.com/2015/01/persistent-mapped-buffers-in-opengl/)

5. **Environment-Aware Motion Matching** (SIGGRAPH Asia 2025)
   - Dynamic environment features in motion matching search
   - Bidirectional pose/trajectory relationship with collision penalization
   - **Source:** [Ponton et al.](https://joseluisponton.com/assets/pdf/emm_siggraphasia2025.pdf)

### Deferred (Vulkan migration)

- **Mesh Shaders** (`GL_NV_mesh_shader`) — NVIDIA-only on OpenGL, AMD support only in Vulkan
- **Sparse Textures** (`GL_ARB_sparse_texture`) — useful for megatextures, not urgent

## Issues Identified but Not Fixed (Require Discussion)

### Architecture

1. **Thread safety** — Logger, EventBus, and InputManager are not thread-safe. Currently
   fine for single-threaded usage but should be documented or guarded if threading is added.

2. **FreeType version** — CVE-2025-27363 is actively exploited. Update via FetchContent
   version pin to >= 2.14.2.

3. **dr_libs version** — Heap overflow in WAV smpl chunk (Issue #296, fixed March 2026).
   Update the committed version.

4. **EventBus unsubscribe** — O(n*m) lookup. Acceptable at current scale but consider
   hash-based lookup if listener counts grow.

5. **Editor size** — `editor.cpp` at 2219 lines. Consider extracting viewport, input
   handling, and render mode logic into separate files.

6. **Test coverage gaps** — Integration tests (physics + animation + cloth), stress tests,
   error injection tests, and fuzzing for asset loaders are not present.

### Shaders

7. **SSAO kernel size** — 64 samples is expensive. Consider XeGTAO or reducing to 32
   with temporal accumulation.

8. **Water shader reverse-Z** — `gl_FragCoord.z` linearization at line 231 should use the
   same reconstruction pattern as `ssao.frag.glsl` for consistency.

9. **GLSL ES compatibility** — No precision qualifiers. Not blocking for desktop OpenGL 4.5
   but prevents any future mobile/WebGL port.
